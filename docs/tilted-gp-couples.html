Tilted GP couples — narrow Fa–Mo via L-bar drops

Addendum to ancestor-refactor.html. The default layout widens parent-couple sep to clear each grandparent couple, leaving an awkward empty corridor between Fa and Mo. This doc describes a tighter layout: every grandparent couple is tilted toward its bloodline child so the inner GP sits directly above the bloodline parent's column, and the GP-Tie midpoint connects to that child via an L-bar drop. Fa and Mo can now sit at the normal COUPLE_PITCH.

Two rules: (1) Each childhoodFamily is built with a tilt = left (bloodline child on the chart's left half — pivot on the GP wife) or right (bloodline child on the right — pivot on the GP husband). The pivoted GP sits at FB-local (0, 0), the other GP at ∓COUPLE_PITCH along the X axis. (2) The FamilyBlock sibship bar now extends to include the child anchor, so a single offset kid connects via an L-bar (drop → horizontal → leg) instead of leaving a gap.

2. The pivot rule

A FamilyBlock placed as a childhoodFamily takes a tilt argument that decides where its FB-local origin (0, 0) lands.

tilt = 'center' // parent FB itself — Fa at −sep/2, Mo at +sep/2 husband at FB-local (−sep/2, 0) wife at FB-local (+sep/2, 0) childAnchorX = 0 // Tie midpoint tilt = 'left' // bloodline child is on chart's left half (Fa side) husband at FB-local (−COUPLE_PITCH, 0) wife at FB-local (0, 0) // wife above bloodline child's column childAnchorX = −COUPLE_PITCH / 2 // Tie midpoint, offset from kid column tilt = 'right' // bloodline child is on chart's right half (Mo side) husband at FB-local (0, 0) // husband above bloodline child's column wife at FB-local (+COUPLE_PITCH, 0) childAnchorX = +COUPLE_PITCH / 2

coupleLeftWidth / coupleRightWidth on PersonBlock keep their existing formula (max(BOX_W/2, childhoodFamily.leftWidth / rightWidth)). With a tilted GP couple, the GP couple's extent is asymmetric — on the inner side it's just BOX_W/2. So the parent FB's sep formula max(COUPLE_PITCH, Fa.coupleRight + Mo.coupleLeft + COUPLE_GAP) naturally produces COUPLE_PITCH without any special casing.

3. L-bar sibship

With a tilted GP couple, the Tie midpoint sits offset from the bloodline kid's column. FamilyBlock's sibship bar now extends to include childAnchorX, so the drop / bar / leg connect cleanly even for a single offset kid.

appendSibshipLines: busY = kidY − ROW_H / 2 drop = vertical from (childAnchorX, childAnchorY) to (childAnchorX, busY) // bar now spans childAnchorX as well as the kid range: minX = min(childAnchorX, kids[*].x) maxX = max(childAnchorX, kids[*].x) if (maxX > minX): bar = horizontal from (minX, busY) to (maxX, busY) legs = vertical from (k.x, busY) to (k.x, kidY − BOX_H/2) for each kid

4. Scope — levels = 2 only (for now)

Tilted GP couples solve the Fa–Mo problem at levels = 2 (default). At levels ≥ 3 the same tilt at depth 2 would put Aapeli's mother and Hilma's father at the same chart X (both want to sit at chart Aapeli's column). That's a separate problem and will need either deeper widening inside the GP couple or a different tilt rule. For now the implementation tilts only at depth 1; depth ≥ 2 keeps the legacy centered behaviour, which means levels = 3 might re-introduce the wide-corridor look one level higher.
