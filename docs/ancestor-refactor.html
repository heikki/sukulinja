Block model — PersonBlock + FamilyBlock

Two block types. FamilyBlock represents exactly one family — its direct members (couple + their kids) and nothing else. PersonBlock represents exactly one person. Recursion (up to ancestors, down to descendants) happens through PersonBlock's own children (childhoodFamily above, marriages below), not by one FamilyBlock containing another.

Membership rule: each FamilyBlock's members are exactly its couple + their kids. Each PersonBlock is drawn once. When a person is "external" to a FB (their PersonBlock lives elsewhere in the tree — e.g., Fa is an adult here but is also a kid in the grandparent FB above), the FB references them for line geometry but does not contain their PersonBlock as a block-tree child.

4. Block-type API

Members vs. block-tree children: each FB knows its full data membership but only renders the PersonBlocks that aren't drawn elsewhere.

class PersonBlock extends Block { readonly personId: number; readonly childhoodFamily: FamilyBlock | null; // placed above as a child of this PB readonly marriages: readonly FamilyBlock[]; // placed below/around; multi-spouse fan rules renderLocal(): draws own box at (0, 0); children: [...marriages..., childhoodFamily (if any)]; } class FamilyBlock extends Block { readonly famId: number; // Full data membership — every person in this family unit readonly members: { husband: { id: number; external: boolean }; // external=true when drawn elsewhere wife: { id: number; external: boolean }; kids: readonly { id: number; external: boolean }[]; }; // Only the non-external members get a PersonBlock here readonly adultBlocks: readonly PersonBlock[]; // 0-2 entries, only for non-external adults readonly kidBlocks: readonly PersonBlock[]; // only for non-external kids renderLocal(): Tie between adult positions (positions resolved via personLocalPos); sibship bar/legs/drop to kid positions; no boxes; children: [...adultBlocks..., ...kidBlocks...]; }

External-member positions are resolved at render time by walking up to the nearest ancestor block that owns the PersonBlock.

The "external" column always points back to the parent FB in this example. In general, the external person's PersonBlock lives wherever the data graph places them; the FB looks up their position via a shared lookup map (or via tree walk).

6. Sibling rendering follows focus

Same FamilyBlock class everywhere. What changes per focus is each kid's PersonBlock: the focused kid has its marriages expanded (with their kids, recursively); everyone else is a bare PersonBlock with empty marriages. Showing aunts/uncles means the grandparent FB's kidBlocks grows from [Fa] to [Fa, Aunt, Uncle].
